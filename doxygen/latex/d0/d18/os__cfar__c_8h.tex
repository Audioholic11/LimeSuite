class {\bf gr\+::radar\+::os\+\_\+cfar\+\_\+c}
\begin{DoxyCompactList}\small\item\em This block estimates peaks of a given F\+FT spectrum as tagged stream. Multi peak detection is implemented with the O\+S-\/\+C\+F\+AR algorithm. The algorithm uses around the cell under test (C\+UT) on each side samp\+\_\+compare samples to estimate the noise floor. This relative threshold is defined by the bin of the vector within the sorted samp\+\_\+compare samples. A standard value is rel\+\_\+threshold = 0.\+78. The value of this bin is multiplied by mult\+\_\+threshold and compared with the C\+UT. samp\+\_\+protect samples are a protected are which is not used for acquiring compare samples. If consecutive bins are detected as valid peaks it is possible to merge these detections with merge\+\_\+consecutive = true. Output data are f32vectors with the information of frequency, power and phase. The identifiers (symbols) are \textquotesingle{}frequency\textquotesingle{}, \textquotesingle{}power\textquotesingle{} and \textquotesingle{}phase\textquotesingle{}. \end{DoxyCompactList}\end{DoxyCompactItemize}
